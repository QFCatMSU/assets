Elements in a webpage

The D2L editor is a web page editor, which means that it is creating HTML code in the background.  While you do not need to know HTML to use this editor, it is important to understand that HTML divides content into elements – and elements can contain other elements. 

Elements is a broad term and it refers to anything in a webpage that can be assigned attributes.

Images, paragraphs, headers, ordered lists, and video are all examples of HTML elements.

Bold, Italics, Underline, and many other font changes are also examples of HTML elements.

When you have a line of text like this.  The words, line of text, are placed inside two HTML elements called Bold and Italics and the whole line is placed in an element called a paragraph.

Element Levels

D2L divide the HTML elements into two main categories:

  2. Line –  elements that take up a whole line (e.g., a paragraph or a header)

  4. inline – elements within a line (e.g., bold, italics, images)

Note: Inline elements are always contained within a Line element.

Attributes

Most of the editing options can be accessed using the tool bar pictured above.  However, many of the more advanced editing options require the Attributes button (fig ##).

There are two main ways to modify elements using Attributes:

  2. Add a Class

    • Class is a predefined set of properties used to modify an element (i.e., the flexSize class added to an image allows it to be resized)

  4. Add an ID

    • IDs are used to link two elements together (i.e., an equation and a reference to the equation)

To add a Class or ID attribute to an element from the editor toolbar:

  2. Click on the element

  4. Click the ( + ) button

  6. Choose Attribute. 

  8. Type a Class or ID in the provided textboxes

  10. Click Create

The Attributes button – this is the button used for most advanced options in the editor

Sectioning the lesson (Headers 1 through 4)

The sections in the lesson are set using headers 1 through 4.

  • Header 1: title of the page only (large blue text – no background color)

  • Header 2: section titles (gray background and large maroon text)

  • Header 3: sub-section titles (gray background and blue text)

  • Header 4: sub-sub-section titles (gray background and green text)

Headers separate the content and create a visual hierarchy.  Headers also improve the accessibility of the page by organizing the HTML to make it easier for accessibility devices to read your page.

Formatting Text within a line (inline elements)

The D2L editor is an HTML editor and HTML is designed around block-level objects (whole line) and inline-level objects (within a line).  The above formats are all block-level.  In this section, we talk about inline objects. 

Inline (fonts) formatting options

Options for formatting fonts

We are editing a webpage, not a document. Font-style changes in a webpage can often have unintended consequences.

D2L gives you the following options (fig ##) to modify content within a line:

  • Bold: good for highlighting content

  • Italics: good for highlighting content (note: italics by itself can often not easily be seen – italics often works better when combined with bold)

  • Underline: in webpages, underlines generally indicate a link, but they can be used for highlighting content

  • Strikethrough (arrow next to Underline): mostly used in drafts to indicate removed content

  • Subscript (arrow next to Underline): useful for formulas

  • Superscript (arrow next to Underline): useful for formulas

You should limit your use of the following inline formats as they can cause problems in a webpage

  • Font Family (dropdown that shows -apple-system): Changes the font type but not all fonts are supported on every platform – it is generally best to stick with the default fonts

  • Size (dropdown that shows 14px): Changes the font size – can cause spacing issues

  • Font Color (button next to Underline): Changing the font color – can cause contrast issues

Highlighting text

If you want to highlight text, it is best to use bold, italics, bold-italics, or one of predefined text formats classes: note and highlight.

Both note and highlight can be inserted using the Attributes command.  Unfortunately, D2L makes this a somewhat byzantine process.

The steps for adding the highlight or note class to text are:

  2. Drag you mouse over the text to select

  4. Click the I button to italicize the text

    • note: this tells D2L that the text is grouped together (it is an inline element)

  6. Click inside the text you just italicized – do not leave the whole text selected

  8. Click down arrow next to the hyperlink and choose Attributes

  10. Type note or highlight to the class textbox

  12. Click Create

If you do not do steps 2 and 3 correctly, D2L will apply text formats to the whole line as opposed to the selected text. If this happens, you can perform an Undo (Control-Z on PC, Command-Z on Mac).

Equation font type, font size, and color

You can only change font type, font-size, and font color within the equation editor.  Changes made using the D2L editor bar will be ignored.

You can use multiple colors in your equation but accessibility is not guaranteed if you do that.  It is probably OK as long as you choose colors that contrast well with the background (e.g., not yellow).

You can only set the font size and font type for the whole equation.

The default font type is Arial and the default font-size is 16px.

Note: the default font size for the document is 14px so it often best to set single-line inline equations (e.g.,e = mc2{“version”:“1.1”,“math”:“<math xmlns=”http://www.w3.org/1998/Math/MathML”><mstyle mathsize=“14px”><mrow><mi>e</mi><mo> </mo><mo>=</mo><mo> </mo><mi>m</mi><msup><mi>c</mi><mn>2</mn></msup></mrow></mstyle></math>“}) to 14px to flow with the rest of the text.

Referencing Element

The referencing element is going to be text that is assigned a Class and ID (in Attributes).  The text can be whatever you choose with the number of the referenced element replace with ##.

Examples:

  • As you can see in figure ##, the …

  • Einstein’s Field Equation (eq. ##) shows that…

  • In section ##, we saw that…

The script will replace ## by the number associated with the referenced element and this number automatically update if the referenced elements number changes.

Most elements in a lesson can be linked to (e.g., code lines, videos, list items) –as long as you give the element an ID.  If the element is not a section, equation, or caption (figure), then the ## will be ignored by the script (so, it is best to not include it).

Otherwise, the above procedures will work to link any element in the page.

Making a reference without the link

If you do not want the referencing element to link to the referenced element then in Attributes, add the Class nolink.

So, the Class will be ref nolink (fig ##)
